____________ Do minuto 24:50 ao 28:11

O projeto curso esta apresentado em uma imagem que esta no link abaixo.

https://forceclass-00.linuxforce.com.br/mod/resource/view.php?id=7853

Esta visão é do que será entregue no final do treinamento, podendo existir variações que podem ser pequenas.

A apresentação de um ambiente virtualizado para criação, configuração dinâmica das máquinas, que simula uma rede interna.

Existem muitos virtualizadores de máquinas, foi instituido um padrão, se existir segurança em administrar estas VMs é permitido o uso no curso, sem necessidade de intervenção, e garantia do aluno que se propõe em usar diferentemente.

Neste cenário o Virtual Box é o virtualizador.

As máquinas virtuais para o projeto estão algumas instaladas, disponíveis, mas sem configurações necessárias e possivelmente incompletas. 

A instalação do Linux esta simples de ser feita atualmente, e até pode ser abordado o processo de instalação posteriormente.

As VMs disponíveis que estão instaladas, vem "cruas", sem nada configurado, pode faltar endereçamento ip, estarem com nomes inadequados, sem serviços

O objetivo do curso Linux Administrator, começa conhecendo como administrar o sistema, depois com outro treinamento chamado de Linux Engineer, onde neste serão implementados e configurados serviços.

____________ Do minuto 28:12 ao 28:11

NOMES DAS MÁQUINAS

Cliente externo: simula uma máquina fora da estrutura, com endereçamento ip específico.

Gateway: desempenha papel de Firewall e Proxy dentro da rede. Inclusive, tem três placas de rede. Com alguns serviços configurados dentro dela.

Intranet: Voltada para servidor Web, DNS.

Datacenter: Um servidor de email, OpenLdap.

Storage: Armazenamento (RAID, LVM, NFS), NGNIX.

Cliente: Windows, implementação de samba.

Cliente Interno: Como notebook do usuário diário, apartir desta conecta-se em outras máquinas da infra estrutura.
